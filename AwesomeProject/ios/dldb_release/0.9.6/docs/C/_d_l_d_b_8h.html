Detailed Description

functions of DLDB SDK

all these functions are synchronous, and thread-safe. no global state is maintained across functions. each function creates its own database connection/handle, and release it when done. all these functions end up writing into the internal sqlite database, so all operations will be serialized, see https://sqlite.org/threadsafe.html. the integrating application must call dldb_init when starting, dldb_heartbeat on a regular low frequency basis. the integrating must call dldb_run_queries_if_any on a frequency compatible with the minimum answering delay expected for queries attached to the dldb_api_key

max_entriesnumber of locations to output, -1 if not limited

outwill point to json string containing the locations log
